Configuration Reference

Neodyme configuration is spread across server.properties (server behaviour) and several JSON files in config/: shop.json, playlists.json, experience.json, game-servers.json, and motd.json. Cloud storage INI files (DefaultGame.ini, DefaultRuntimeOptions.ini) are documented on the Game Files page. No restart is required for shop changes when using /shop rotate force; most server.properties changes require a restart.

config/shop.json

Controls the automatic item shop rotation. Changes take effect immediately when you run /shop rotate force from the server console - no restart needed.

KeyDefaultDescription
randomShopfalseWhen true, the shop is generated on-the-fly per request from the fortnite-api.com cosmetics cache - data/shop.json is never read or written. Chapter 1/2 layout is auto-detected from the connecting client's version: Season ≤ 10 -> Ch1 (4 daily + 2 featured, BRDailyStorefront / BRWeeklyStorefront) ; Season > 10 -> Ch2+ (counts from shopCategories). Cosmetics are filtered to the client's own era (backendValue ≤ season), so a Season 5 client only sees Season 1-5 items while a Season 14 client sees Seasons 1-14 items. The shop is seeded on season + date - identical for the same client reconnecting the same day, automatically different the next day. isChapter1, shopRotationTime, and the rotation scheduler are all ignored when this mode is active.
isChapter1falseWhen true, forces Chapter 1 shop mode: the shop always uses exactly two fixed storefronts - BRDailyStorefront (4 small tiles, daily rotation) and BRWeeklyStorefront (2 normal tiles, weekly rotation). The shopCategories block is ignored in this mode. Set to true when hosting Chapter 1 builds (Seasons 1-10). Ignored when randomShop: true (Chapter detection is automatic).
shopRotationTime"00:00"Time of day (HH:MM, 24-hour) at which the daily rotation fires automatically. Requires autoShopRotation=true in server.properties. Unused when randomShop: true.
shopMaxSeason32Maximum backendValue of cosmetics to include in the shop. 0 = no limit (all seasons). See the reference table below for values per season. Ignored when customVersion=false (era is derived from fnVersion major component instead). In randomShop mode the client's own season is used as the ceiling - this setting is ignored.
shopExcludedItemsarrayArray of cosmetic IDs to never include in the shop, regardless of other filters. Battle Pass and Winterfest reward items are always excluded automatically (read from content/athena/battlepasses/ and content/athena/winterfest-rewards.json). Applied in both normal and randomShop mode.
includeBattlePasstrueInclude the Battle Pass bundle in the season storefront (BRSeason<N>). Set to false to hide the Battle Pass offer from the catalog.
shopCategoriesobjectDefines each shop section. Ignored when isChapter1: true or randomShop: true. See below.

shopCategories

Each key under shopCategories is a section identifier used as the prefix for shop item keys in data/shop.json (e.g. key "daily" -> items daily1, daily2...). The built-in sections are daily and featured. You can add custom sections with any lowercase name.

FieldDescription
countNumber of items to pick randomly for this section each rotation.
displayNameDisplay label shown in the game UI.
storefrontNameInternal storefront identifier sent to the game client. Do not change unless you know the client-side storefront name.
tileSizeTile size in the shop UI: "Small" (daily) or "Normal" (featured).
rotationInterval"daily" or "weekly". Daily sections rotate every day at shopRotationTime; weekly sections rotate every 7 days.

shopMaxSeason - backendValue reference

How it works: Every cosmetic from the fortnite-api.com dataset has an introduction.backendValue - a monotonically increasing integer that identifies the season the item was introduced. Setting shopMaxSeason to a value N means only items with backendValue <= N can appear in the shop. The value is inclusive - setting 13 will include Chapter 2 Season 3 items.
backendValueSeasonbackendValueSeason
1Chapter 1 Season 117Chapter 2 Season 7
2Chapter 1 Season 218Chapter 2 Season 8
3Chapter 1 Season 319Chapter 3 Season 1
4Chapter 1 Season 420Chapter 3 Season 2
5Chapter 1 Season 521Chapter 3 Season 3
6Chapter 1 Season 622Chapter 3 Season 4
7Chapter 1 Season 723Chapter 4 Season 1
8Chapter 1 Season 824Chapter 4 Season 2
9Chapter 1 Season 925Chapter 4 Season 3
10Chapter 1 Season X26Chapter 4 Season 4
11Chapter 2 Season 127Chapter 4 Season 5 (OG)
12Chapter 2 Season 228Chapter 5 Season 1
13Chapter 2 Season 329Chapter 5 Season 2
14Chapter 2 Season 430Chapter 5 Season 3
15Chapter 2 Season 531Chapter 5 Season 4
16Chapter 2 Season 632Chapter 5 Season 5 (REMIX)

Example: to serve a classic Chapter 1 experience, set "shopMaxSeason": 10. To include all seasons through Chapter 2 Season 3, set "shopMaxSeason": 13 (13 is inclusive).

customVersion=false mode: When customVersion=false, the shopMaxSeason field is ignored. Instead, the major component of fnVersion is used as the limit (e.g. fnVersion=7.40 -> shopMaxSeason=7).
