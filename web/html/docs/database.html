Database

Neodyme supports multiple database backends. All data access goes through the DatabaseManager abstraction layer, ensuring consistent behavior regardless of the storage engine.

Supported Databases

JSON Database (Default)

The simplest option. All data is stored in JSON files in the data/ directory.

Configuration

databaseType=json
databasePath=data/
getJsonSpacing=true    # Pretty-print for readability

Data Files

data/
  clients.json          # User accounts
  friends.json          # Friend relationships
  tickets.json          # Support tickets
  creator-codes.json    # Creator codes and requests
  audit-log.json        # Audit log entries
  reports.json          # User reports

Characteristics

  • No external dependencies
  • Human-readable data files
  • Atomic writes with temp files for safety
  • Suitable for small to medium servers

MongoDB

Production-ready database with high scalability.

Setup

  1. Install MongoDB on your system or use MongoDB Atlas (cloud)
  2. Install the Node.js driver: npm install mongodb mongoose
  3. Configure server.properties:
databaseType=mongodb
databasePath=mongodb://localhost:27017/neodyme

Connection String Examples

# Local MongoDB
databasePath=mongodb://localhost:27017/neodyme

# MongoDB with authentication
databasePath=mongodb://user:password@localhost:27017/neodyme

# MongoDB Atlas (cloud)
databasePath=mongodb+srv://user:password@cluster.mongodb.net/neodyme

Architecture

All database operations go through DatabaseManager, which delegates to the active implementation. Switching databases only requires changing databaseType in server.properties.

Service Layer (TicketService, AuditService, ...)
       |
  DatabaseManager (abstraction)
       |
  JsonDatabase  OR  MongoDatabase

Data Migration NEW 1.2.4

Migrate data between database backends using the /data command:

# Preview what would be migrated (no data written)
/data migrate json mongodb --dry-run

# Migrate from JSON to MongoDB
/data migrate json mongodb --confirm

# Migrate from MongoDB back to JSON
/data migrate mongodb json --confirm

Migration Flags

FlagDescription
--dry-runPreview migration - no data is written
--confirmSkip the confirmation prompt and run immediately

Backup System

Configuration

databaseBackup=true              # Enable backups
databaseBackupInterval=60        # Backup every 60 minutes
databaseBackupExpiryDays=7       # Delete backups older than 7 days

Commands

/backup create              # Create manual backup
/backup list                # List available backups
/backup stats               # Backup statistics
/backup restore <name>      # Restore a backup (requires /confirm restore)
/backup delete <name>       # Delete a backup (requires /confirm delete)
/backup cleanup             # Force cleanup of expired backups

Restore Process

  1. Run /backup restore <backup-name>
  2. Confirm with /confirm restore
  3. Restart the server after restoration

Backups are stored in data/backups/ with timestamped folder names.

Redis Cache (Optional)

Redis can be used alongside any database for caching tokens, profiles, and sessions.

Configuration

redisEnabled=false
redisHost=127.0.0.1
redisPort=6379
redisPassword=

Setup

npm install ioredis

Redis improves performance by reducing database queries on frequently accessed data (auth tokens, profile data, session info).
