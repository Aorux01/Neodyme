Admin Panel

The Admin Panel is reserved for trusted operators. It covers user management (role changes, permanent bans), the full server-wide audit log with full diffs, detailed statistics with historical charts, and live web/security settings. Access requires the admin or owner role.

With great power… — Admins can change roles, permanently ban accounts, and modify live server settings. There is no "undo" on most of these — the audit log is your safety net. Use the Audit Log to review your own actions periodically.

1. User Management

The Users tab is a searchable table of every account, paginated. For each user:

  • Display name, email, accountId, created at, last login
  • Current role (Player / Founder / Helper / Moderator / Developer / Admin / Owner / Server)
  • Ban status (banned + reason + duration + expiry, or not banned)

Change role

Dropdown next to each user. Promoting someone to admin or owner gives them access to this panel — do it consciously. The change is logged with change_role (old role → new role).

Role hierarchy from least to most privileged:

  1. player — default for new sign-ups
  2. founder — cosmetic-only (founder pack)
  3. helper — cosmetic + read-only on tickets (no enforcement powers)
  4. moderator — tickets, reports, bans, creator codes
  5. developer — full Developer Panel (config, content, assets, shop…)
  6. admin — everything moderators have + this panel
  7. owner — admin + cannot be demoted by another admin
  8. server — reserved for dedicated-server host accounts (Host0001, …). Don't assign to humans.

Ban a user

Same modal as the Moderation Panel (reason + duration + invalidate sessions). Admins can apply permanent bans that auto-expire never.

Unban

Available on banned users. Justification required, logged.

2. Audit Log

The audit log is the most important admin tool. It records every staff action that modifies state — nothing is silent.

Header row (always visible)

  • Action badge (colour-coded icon + label, e.g. Ban User, Edit Content, Randomize Slot)
  • Who — display name of the actor
  • Target — the affected resource (user:<accountId>, content:motd:root, shop_slot:daily3…)
  • IP address (small, right-aligned)
  • Time — HH:MM:SS + date

Detail panel (click to expand)

Every entry's full details payload is rendered as a key-value table. The new audit modal pays special attention to before/after diffs:

  • Diff pairs — when both oldValue/newValue, oldRole/newRole, oldPrice/newPrice, oldItemGrants/newItemGrants are present, they're rendered side-by-side: WAS  old  →  NOW  new
  • Booleans render in green ("yes") / red ("no")
  • Arrays render inline with overflow truncation past 12 items
  • Small objects render as inline JSON; large objects get a scrollable <pre> block
  • Fields already visible in the header (scope, section, slot, path) are hidden in the detail to avoid duplication

Filters & search

Above the log:

  • Action filter (dropdown of all action types)
  • Search (matches actor name, target ID, action name, ban reason, target user name)
  • Date range — from + to
  • Pagination — 25 entries per page, "Page N / M"

24h summary bar

Above the filters, a chip strip shows the top-5 most-frequent action types in the last 24 hours, with counts. Lets you spot anomalies (e.g. 200 bans in a day = someone went rogue or a wave of cheaters).

Audit log storage

Audit entries live in data/audit-log.json (JSON adapter) or the auditlogs collection (Mongo adapter). They are never deleted automatically — only manual pruning via /audit clean <days> from the console.

3. Statistics

The Statistics tab shows:

  • Total accounts, active players (last 5 min), peak concurrent
  • Player history chart — 24h / 7d / 30d toggles
  • System health history — pass rate of the System Tests over the last 72h

The underlying samples are recorded by player-stats-service every 5 minutes.

4. Web Settings

Quick-access toggles for the most-tweaked production settings. Each toggle saves immediately:

  • Rate Limiting — master toggle for the global IP limiter
  • Max login attempts — per IP, before lockout (default 5)
  • Lockout window (minutes) — how long the lockout lasts (default 15)
  • CORS, Helmet, Compression, Trust Proxy — security middleware toggles

These map to the same server.properties keys as the Developer Panel's Configuration tab — this tab is just a curated shortcut.

5. Commands

Admin-only console shortcuts:

  • Database: backup, migrate JSON↔Mongo, restore from backup
  • Accounts: bulk role change, bulk unban, prune inactive (with --dry-run)
  • Logs: tail server logs, clear logs

Security model

  • All admin routes require requireAdmin middleware (role gate)
  • Writes additionally require CSRF (cookie neodyme_session + header X-CSRF-Token) — provided automatically by secureFetch() in the frontend
  • Every write hits the audit log before the response is sent — you cannot silently change state
  • Admins cannot demote themselves below admin (the API refuses to avoid lockout)
  • Only owner can demote an admin; an admin cannot demote another admin
