Version 1.2.0 - Release

January 31, 2026
RELEASE

Critical security hotfixes and enhanced server architecture.

  • Bcrypt Work Factor: Increased to 12-14 (configurable) for 2026 security standards
  • Password Complexity: Enforced requirements (uppercase, lowercase, numbers, special chars)
  • Timing Attack Protection: Random delays (50-200ms) prevent account enumeration
  • Token Encryption: Optional AES-256-GCM encryption for tokens at rest
  • Session Management: Token rotation with family detection, IP binding, device fingerprinting
  • Session Dashboard: New "Security & Sessions" tab to view and manage active sessions
  • Global Logout: "Log Out All Devices" functionality added
  • Request Limits: Configurable 1MB limit for regular requests (50MB for cloud storage)
  • Shop Categories: Support for custom shop categories beyond Daily/Featured
  • V-Bucks API: New dedicated endpoints for V-Bucks management
  • EOS Connect: The server can now handle the connection correctly
  • Server architecture: The server architecture was reviewed and cleaned up
  • Admin Panel: The server now has an administrator panel

Version 1.1.6 - Security Hardening

January 15, 2026
SECURITY

Critical security improvements and Redis support.

  • Token Hashing: Tokens are now stored as SHA-256 hashes - raw tokens never persisted
  • JWT Secret Rotation: Automatic monthly rotation with 256-bit secrets
  • HttpOnly Cookies: Authentication tokens moved from localStorage to secure HttpOnly cookies with SameSite=Strict
  • CSRF Protection: Added CSRF middleware with token validation for all forms
  • Account Enumeration Fix: /check-email and /check-username now return uniform responses
  • Redis Support: Optional Redis storage for tokens and cache (configurable in server.properties)
  • Async File Operations: Converted synchronous file reads to async for better performance
  • Site: Website functionality improvements

Version 1.1.4 - Security Hardening

December 13, 2025
SECURITY

Major security fixes addressing 10 critical vulnerabilities.

  • Token Persistence: Tokens now persist to Tokens.json file instead of in-versionInfo storage - survives server restarts
  • Input Validation: Added comprehensive input validation for all authentication endpoints with length limits
  • Password Requirements: Enforced strong password policy (8+ chars, uppercase, lowercase, number, special char) with visual strength indicator
  • Email Normalization: All emails are now normalized to lowercase to prevent duplicate accounts
  • Account Lockout: Added account lockout after 5 failed login attempts with exponential backoff
  • Race Condition Protection: Implemented file locking and atomic write operations for all JSON database operations
  • Shop Images: Shop now displays item images from fortnite-api.com and shows "Already Owned" badge
  • Token Commands: New /tokens, /health, /ready, /unlock commands for token and server management
  • Website: Updated the Neodyme website
